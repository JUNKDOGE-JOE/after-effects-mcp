## Remaining gates

- The Provider v1 protocol still has a narrow hard-crash ambiguity: the Helper can
  commit a newly created secret while both the set response and recovery read are
  unavailable. The current implementation never blind-deletes. Full durability
  requires a protocol operation ID plus a redacted create-intent journal and
  startup reconciliation.
- The macOS identity policy distinguishes the Adobe-signed CEP process and its AE
  ancestry, not one extension instance from another same-user Adobe CEP extension.
  This residual threat must be accepted for the development smoke milestone or
  replaced by explicit pairing/companion-app attestation.
- The installed stable AE 2026 application currently fails strict Adobe code-sign
  verification and Creative Cloud offers an update. Do not update or close AE while
  an unsaved project may exist; obtain action-time confirmation before installation.
- Windows CI, PowerShell parsing, the x64 N-API/Named Pipe path, Authenticode, and AE
  25/26 smoke remain Windows handoff work.
